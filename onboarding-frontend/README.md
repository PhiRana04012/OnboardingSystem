# Система онбординга - Frontend

Веб-приложение для системы онбординга новых сотрудников, построенное на Vue.js 3.

## Технологии

- **Vue 3** - Progressive JavaScript Framework
- **Vue Router** - Официальный маршрутизатор для Vue.js
- **Pinia** - State management для Vue
- **Vite** - Сборщик и dev-сервер
- **Tailwind CSS** - Utility-first CSS framework
- **Axios** - HTTP клиент

## Установка и запуск

### Предварительные требования

- Node.js 18+ 
- npm или yarn

### Установка зависимостей

```bash
npm install
```

### Запуск dev-сервера

```bash
npm run dev
```

Приложение будет доступно по адресу `http://localhost:3000`

### Сборка для production

```bash
npm run build
```

Собранные файлы будут в папке `dist/`

### Просмотр production сборки

```bash
npm run preview
```

## Конфигурация

### API Endpoint

По умолчанию приложение настроено на работу с бэкендом на `http://localhost:5000`. Это настраивается в `vite.config.js` через proxy:

```javascript
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:5000',
      changeOrigin: true
    }
  }
}
```

### Аутентификация

В текущей версии реализована упрощённая аутентификация через ID пользователя. В продакшене необходимо интегрировать SSO (Single Sign-On).

## Функциональность

### Роли пользователей

- **Новый сотрудник** - проходит модули и тесты
- **Наставник** - видит прогресс подопечных
- **HR-специалист** - управляет пользователями и формирует отчёты
- **Руководитель подразделения** - видит отчёты по своему подразделению
- **Администратор системы** - полный доступ к управлению системой

### Основные функции

1. **Дашборд** - главная страница с прогресс-баром и списком модулей
2. **Просмотр модулей** - изучение материалов модуля
3. **Прохождение тестов** - тестирование знаний по модулям
4. **Отчёты** - различные отчёты для разных ролей
5. **Администрирование** - управление модулями, пользователями и вопросами

## Интеграция с бэкендом

Приложение использует REST API бэкенда. Все API методы определены в `src/api/services.js`:

- `usersApi` - управление пользователями
- `modulesApi` - управление модулями
- `progressApi` - прогресс онбординга
- `questionsApi` - управление вопросами
- `testAttemptsApi` - прохождение тестов
- `reportsApi` - формирование отчётов
- `departmentsApi` - управление подразделениями

## Стилизация

Приложение использует Tailwind CSS для стилизации. Основные классы определены в `src/style.css`:

- `.btn-primary` - основная кнопка
- `.btn-secondary` - вторичная кнопка
- `.card` - карточка контента
- `.input` - поле ввода

Цветовая схема настраивается в `tailwind.config.js`.
